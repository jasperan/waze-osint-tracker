01 / Architecture

Intelligence Module Connectivity

Before: 12+ modules computing sophisticated analysis with zero visualization. After: every module wired to interactive frontend.

Backend Modules (already existed)

  • social_graph.py — force graph, communities, ego networks
  • encounter_prediction.py — meeting probability, hotspots
  • anomaly_detection.py — time/location/frequency scoring
  • temporal_fingerprint.py — 168-bin weekly histograms
  • privacy_score.py — 6-subscore risk assessment
  • trip_reconstruction.py — event-to-trip segmentation
  • geofence.py — zone-based containment
  • intel_vectors.py — 44-dim behavioral fingerprints
  • intel_routines.py — DBSCAN home/work clustering
  • intel_combined.py — vector + graph fusion

New Frontend Layers (this release)

  • social-graph.js — D3.js force-directed graph widget
  • encounter-map.js — Leaflet heatmap with time slider
  • anomaly-feed.js — SSE-connected real-time alert stream
  • dossier_builder.py — fusion orchestrator for all modules
  • dossier.html — self-contained Jinja2 report template
  • anomaly_feed.py — micro-batch streaming adapter

Zero new Python dependencies. All frontend via CDN (D3.js, Leaflet, Chart.js). No database schema changes.

02 / Feature
🔗

Interactive Social Graph

D3.js force-directed graph with community detection and ego networks

Capabilities

  • Nodes colored by community (label propagation detection)
  • Node size scaled by event count (sqrt scale)
  • Edge thickness by co-occurrence weight
  • Edge color by relationship type (CONVOY, SAME_PERSON, etc.)
  • Click any node to zoom into ego network (2-hop BFS)
  • Tooltip: username, event count, top report type, community ID
  • 200-node performance guard with "load more"
  • ResizeObserver for responsive container fitting
GET /api/social-graph?limit=200
GET /api/social-graph/<username>
03 / Feature
📍

Encounter Prediction Map

Leaflet heatmap overlay with 168-bin temporal controls

Capabilities

  • Heatmap intensity = encounter probability (0-1)
  • Day-of-week selector (Mon-Sun or All)
  • Hour slider (0-23 or All) for temporal filtering
  • Click hotspot markers for: users, probability %, evidence
  • Toggle visibility on/off without destroying data
  • Wired to encounter_prediction + temporal_fingerprint modules
GET /api/encounters/schedule?day=0&hour=8&limit=500
GET /api/encounters/hotspots
GET /api/encounters/<user_a>/<user_b>
04 / Feature
⚡

Real-time Anomaly Feed

SSE-streamed anomaly detection with geofence integration

Architecture

  • Micro-batch adapter (not per-event) in status_monitor_thread
  • Sliding window deque(maxlen=1000) for baseline statistics
  • Cross-references GeofenceManager for zone containment
  • Broadcasts via existing SSE infrastructure (new "anomaly" event type)
  • Three anomaly types: time, location, frequency

Severity Coloring

Score < 30 30-70 Score > 70
GET /api/anomalies?limit=100
SSE /api/events/stream → type: "anomaly"

OSINT Dossier Export

Self-contained HTML intelligence report fusing all modules

Report Sections

  • Header with username, timestamp, event count
  • Privacy Risk Score with 6 sub-score gauges
  • Activity Profile (event types, active days, area)
  • Location Intelligence (home/work/transit inference)
  • Trip History with distance and duration
  • Social Network (top 10 connections)
  • Temporal Fingerprint (7x24 activity heatmap)
  • Anomaly Report with time/location/frequency
  • AI Narrative (Ollama, graceful fallback)
